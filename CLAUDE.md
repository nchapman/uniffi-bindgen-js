## Project Overview

uniffi-bindgen-js is a JavaScript/TypeScript backend for UniFFI ŌĆö it generates idiomatic JS/TS bindings that call UniFFI FFI functions directly. Like the first-party UniFFI backends (Kotlin, Swift, Python), users compile their Rust crate to WASM and run one command to get bindings. No wasm-pack, no wasm-bindgen.

```bash
cargo build --target wasm32-unknown-unknown --release
uniffi-bindgen-js generate target/.../foo.wasm --out-dir ./bindings
```

## Architecture

### Crate Layout

- **`crates/ubjs_bindgen`** (`uniffi-bindgen-js`) ŌĆö Main code generator. CLI entrypoint, config parsing, JS/TS output.
- **`crates/ubjs_runtime`** (`uniffi-bindgen-js-runtime`) ŌĆö Runtime support library for generated JS bindings.
- **`crates/ubjs_testing`** (`uniffi-bindgen-js-testing`) ŌĆö Shared test utilities for fixture and integration testing.

### Generator Pipeline

```
main.rs ŌåÆ cli.rs (Clap) ŌåÆ lib.rs::run() ŌåÆ js/mod.rs::generate_bindings()
```

`js/mod.rs` is the core generation module. It resolves config from an optional TOML file (`[bindings.js]` section), derives the module name (PascalCase from UDL filename), and renders TypeScript output that calls UniFFI FFI functions directly via the FFI buffer convention.

### Generated Output Format

The generator produces TypeScript that calls UniFFI FFI functions directly:

```typescript
// Generated by uniffi-bindgen-js. DO NOT EDIT.
import { UniffiRuntime, UniFFIWriter, UniFFIReader } from './uniffi_runtime.js';

const _rt = await UniffiRuntime.load(new URL('./simple_fns.wasm', import.meta.url), 'simple_fns');

export namespace SimpleFns {
  export function greet(name: string): string {
    const _rb_name = _rt.lowerString(name);
    const _argPtr = _rt.scratchAlloc(3 * 8);
    _rt.writeRustBufferElements(_argPtr, _rb_name);
    const _retPtr = _rt.scratchAlloc(7 * 8);
    _rt.call('uniffi_ffibuffer_simple_fns_fn_func_greet', _argPtr, _retPtr);
    _rt.checkCallStatus(_retPtr + 24);
    const _result = _rt.liftString(_rt.readRustBufferElements(_retPtr));
    _rt.scratchReset();
    return _result;
  }
}
```

Requires `uniffi = { features = ["scaffolding-ffi-buffer-fns", "wasm-unstable-single-threaded"] }` in the Rust crate.

### Supported Input Sources

- **`.wasm` file**: Extract metadata directly from compiled WASM, copy `.wasm` to output dir
- **`.udl` file**: Parse UDL, generate FFI-direct output (user provides `.wasm` at runtime)
- **`.dylib/.so/.dll`**: Library mode, extract metadata from native cdylib

### Golden Test Pattern

Fixtures live in `fixtures/{name}/` with UDL input at `src/{name}.udl` and expected output at `expected/{name}.ts`. The test harness (`crates/ubjs_bindgen/tests/golden_generated.rs`) generates bindings and compares byte-for-byte against expected files. When changing generator output, update the corresponding expected files.

Additionally, `scripts/typecheck_golden.sh` (or `just typecheck-golden`) runs `tsc --noEmit` on all golden files with auto-generated `.d.ts` stubs for the `uniffi_runtime.js` import, catching type errors that byte-for-byte comparison alone would miss.

### Naming Conventions

- Generated module names use PascalCase (e.g., `simple_fns` ŌåÆ `SimpleFns`)
- Exported TypeScript names use camelCase (e.g., `broken_greet` ŌåÆ `brokenGreet`)
- FFI function names use `uniffi_ffibuffer_{ns}_fn_{kind}_{name}` pattern
- UniFFI uses `.to_ascii_lowercase()` for object names in FFI exports (NOT snake_case)
- Output files are lowercase `.ts` (e.g., `simple_fns.ts`)
- Published crate: `uniffi-bindgen-js` (lib name: `uniffi_bindgen_js`)

### External Type Support

External types are declared in UDL with `[External="crate_name"]` and configured in `uniffi.toml`:
```toml
[bindings.js]
external_packages = { other_crate = "./other_bindings.js" }
```
The generator emits named imports and errors if a used external crate has no config entry.

## Development Methodology

- **TDD-first**: Write failing tests before implementation for each UDL feature.
- **Idiomatic output**: Generated code must look like hand-written JS/TS, not Rust transliterations. It should pass Prettier and the TypeScript compiler.
- **Deterministic generation**: Golden tests enforce reproducible output.
- **Linear history**: Keep commits as coherent units with descriptive messages in imperative mood.
