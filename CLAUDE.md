## Project Overview

uniffi-bindgen-js is a JavaScript/TypeScript backend for UniFFI — it generates idiomatic JS/TS bindings from UDL definitions. Built in Rust, it generates TypeScript wrappers around wasm-pack output, enabling Rust code compiled to WebAssembly to be called from JavaScript/TypeScript universally (Node.js, browsers, Deno, Bun).

## Build & Test Commands

```bash
# Build Rust workspace
cargo build --workspace

# Run Rust tests (unit + golden)
cargo test --workspace

# Full CI check (run before committing)
cargo fmt --check && cargo clippy --all-targets -- -D warnings && cargo test --workspace

# Build WASM fixture and JS bindings, then run all tests (Rust + JS smoke tests)
./scripts/test_bindings.sh

# Build bindings only (WASM + generator + pnpm install)
./scripts/build_bindings.sh

# Generate bindings from a UDL file
cargo run --bin uniffi-bindgen-js -- generate fixtures/simple-fns/src/simple_fns.udl --out-dir /tmp/out

# Run a single Rust test
cargo test -p ubjs_bindgen golden_simple_fixture
cargo test -p ubjs_bindgen pascal_case

# Run JS smoke tests (requires generated/ artifacts from build_bindings.sh)
cd binding_tests && pnpm test
```

## Architecture

### Crate Layout

- **`crates/ubjs_bindgen`** — Main code generator. CLI entrypoint, config parsing, JS/TS output.
- **`crates/ubjs_runtime`** — Runtime support library (scaffold, not yet implemented).
- **`crates/ubjs_testing`** — Test utilities (scaffold, not yet implemented).

### Generator Pipeline

```
main.rs → cli.rs (Clap) → lib.rs::run() → js/mod.rs::generate_bindings()
```

`js/mod.rs` is the core generation module. It resolves config from an optional TOML file (`[bindings.js]` section), derives the module name (PascalCase from UDL filename), and renders TypeScript output.

### Generated Output Format

The generator produces a thin TypeScript wrapper around the wasm-pack JS glue:

```typescript
// Generated by uniffi-bindgen-js. DO NOT EDIT.
import __init, * as __bg from './simple_fns_bg.js';
export { __init as init };
// (external type imports here, if external_packages configured)

export namespace SimpleFns {
  export function greet(name: string): string { return __bg.greet(name); }
}
```

**Convention**: wasm-pack is built with `--target web --out-name {namespace}_bg`, producing `{namespace}_bg.js` (default async `init` export + all wasm functions) and `{namespace}_bg_bg.wasm`. The generated `{namespace}.ts` co-exists with these files without naming conflicts.

**wasm-bindgen naming**: Top-level functions keep their Rust snake_case names in the JS glue (`__bg.my_fn`). Object methods also keep their Rust snake_case names in the JS glue (`this._inner.my_method`). The generator applies camelCase only to the *exported* TypeScript names, not the inner wasm-bindgen calls.

### WASM Build Flow

For each fixture with a wasm crate (e.g., `fixtures/simple-fns/wasm/`):
1. `wasm-pack build --target web --out-name {namespace}_bg --out-dir {out_dir}` — compiles Rust to WASM, emits `{namespace}_bg.js` and `{namespace}_bg_bg.wasm`
2. `uniffi-bindgen-js generate` — reads the UDL and emits `{namespace}.ts`
3. Consumer calls `await init({ module_or_path: wasmBytes })` before using the namespace

### Golden Test Pattern

Fixtures live in `fixtures/{name}/` with UDL input at `src/{name}.udl` and expected output at `expected/{name}.ts`. The test harness (`crates/ubjs_bindgen/tests/golden_generated.rs`) generates bindings and compares byte-for-byte against expected files. When changing generator output, update the corresponding expected files.

### Naming Conventions

- Generated module names use PascalCase (e.g., `simple_fns` → `SimpleFns`)
- Exported TypeScript names use camelCase (e.g., `broken_greet` → `brokenGreet`)
- Inner wasm-bindgen calls use the original Rust snake_case name (e.g., `this._inner.reset_to(v)`)
- Output files are lowercase `.ts` (e.g., `simple_fns.ts`)
- Crate names prefixed with `ubjs_`

### External Type Support

External types are declared in UDL with `[External="crate_name"]` and configured in `uniffi.toml`:
```toml
[bindings.js]
external_packages = { other_crate = "./other_bindings.js" }
```
The generator emits named imports and errors if a used external crate has no config entry.

## Development Methodology

- **TDD-first**: Write failing tests before implementation for each UDL feature.
- **Idiomatic output**: Generated code must look like hand-written JS/TS, not Rust transliterations. It should pass Prettier and the TypeScript compiler.
- **Deterministic generation**: Golden tests enforce reproducible output.
- **Linear history**: Keep commits as coherent units with descriptive messages in imperative mood.
