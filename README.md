# uniffi-bindgen-js

Call Rust code from JavaScript and TypeScript.

[![Crates.io](https://img.shields.io/crates/v/uniffi-bindgen-js)](https://crates.io/crates/uniffi-bindgen-js)
[![CI](https://github.com/nchapman/uniffi-bindgen-js/actions/workflows/ci.yml/badge.svg)](https://github.com/nchapman/uniffi-bindgen-js/actions/workflows/ci.yml)
[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](https://opensource.org/licenses/MIT)

`uniffi-bindgen-js` is a third-party [UniFFI](https://mozilla.github.io/uniffi-rs/) backend that generates idiomatic TypeScript bindings from Rust, targeting WebAssembly via [wasm-pack](https://rustwasm.github.io/wasm-pack/). Define your API once in a UniFFI interface definition, and get typed, documented TypeScript that works in browsers, Node.js, Deno, and Bun.

## Quickstart

**1. Define your interface** in a `.udl` file:

```webidl
// greet.udl
namespace greet {};

string greet(string name);
```

**2. Implement it in Rust** with a `#[wasm_bindgen]` crate (see the [wasm-pack docs](https://rustwasm.github.io/wasm-pack/) for project setup).

**3. Build the WASM module** with wasm-pack:

```bash
wasm-pack build --target web --out-name greet_bg --out-dir pkg/
```

> The `--out-name` must be `{namespace}_bg`. This produces `greet_bg.js` and `greet_bg_bg.wasm`. The double `_bg` in the `.wasm` filename is expected -- wasm-pack appends `_bg.wasm` to whatever `--out-name` you give it.

**4. Generate TypeScript bindings:**

```bash
uniffi-bindgen-js generate greet.udl --out-dir pkg/
```

**5. Generated output** (`greet.ts`):

```typescript
// Generated by uniffi-bindgen-js. DO NOT EDIT.
import __init, * as __bg from './greet_bg.js';
export { __init as init };

export namespace Greet {
  export function greet(name: string): string { return __bg.greet(name); }
}
```

**6. Use it:**

```typescript
import { init, Greet } from './pkg/greet.js';

// Initialize the WASM module once before use
await init({ module_or_path: wasmBytes });

console.log(Greet.greet('world')); // "Hello, world!"
```

## Install

Requires [Rust](https://rustup.rs/) and [wasm-pack](https://rustwasm.github.io/wasm-pack/installer/).

```bash
cargo install uniffi-bindgen-js
```

Or build from source:

```bash
git clone https://github.com/nchapman/uniffi-bindgen-js
cd uniffi-bindgen-js
cargo build --release
```

## What it generates

Generated TypeScript is designed to look like hand-written code. Exported names use camelCase, while internal calls to the wasm-bindgen glue layer retain the original Rust snake_case names.

### Functions

Top-level functions are grouped into a namespace named after the UDL file (PascalCase).

```typescript
import __init, * as __bg from './simple_fns_bg.js';
export { __init as init };

export namespace SimpleFns {
  export function greet(name: string): string { return __bg.greet(name); }
  export async function greetAsync(name: string): Promise<string> { return await __bg.greet_async(name); }
  export function greetOptional(name: string | null = null): string { return __bg.greet_optional(name); }
}
```

### Objects

UniFFI `interface` types become TypeScript classes with constructor, methods, lifecycle management, and `Symbol.dispose` support.

```typescript
export class Counter {
  private readonly _inner: __bg.Counter;
  private _freed = false;
  private _assertLive(): void {
    if (this._freed) throw new Error('Counter object has been freed');
  }
  private constructor(inner: __bg.Counter) { this._inner = inner; }
  static create(start: bigint): Counter { return Counter._fromInner(new __bg.Counter(start)); }
  increment(): void { this._assertLive(); this._inner.increment(); }
  get(): bigint { this._assertLive(); return this._inner.get(); }
  resetTo(value: bigint): void { this._assertLive(); this._inner.reset_to(value); }
  free(): void {
    if (this._freed) return;
    this._freed = true;
    this._inner.free();
  }
  [Symbol.dispose](): void { this.free(); }
}
```

### Records, enums, and data enums

```typescript
export interface Point {
  x: number;
  y: number;
}

export type Direction = 'North' | 'South' | 'East' | 'West';

export type Shape =
  | { tag: 'Circle', radius: number }
  | { tag: 'Rectangle', width: number, height: number }
  | { tag: 'Point' };
```

## Usage

### Generate from a UDL file

```bash
uniffi-bindgen-js generate path/to/definitions.udl --out-dir out/
```

### Generate from a compiled library (library mode)

```bash
uniffi-bindgen-js generate path/to/libmycrate.so --out-dir out/
```

Use `--crate <name>` to generate bindings for a specific crate only.

### Configuration

Place a `[bindings.js]` section in your `uniffi.toml` and pass it with `--config`:

```toml
[bindings.js]
module_name = "MyBindings"
rename = { add_numbers = "sumValues", "Counter.currentValue" = "getValue" }
exclude = ["internal_helper"]
external_packages = { other_crate = "./other_bindings.js" }
```

See [docs/configuration.md](docs/configuration.md) for the full reference.

### External types

External types declared with `[External="crate_name"]` in UDL require a corresponding entry in `external_packages`:

```toml
[bindings.js]
external_packages = { other_crate = "./other_bindings.js" }
```

The generator emits named imports from the configured path.

## Loading WASM

The generated `init()` function must be called once before using any bindings. How you load the `.wasm` file depends on your environment:

**Node.js:**

```typescript
import { readFile } from 'node:fs/promises';
import { init, MyModule } from './my_module.js';

const wasm = await readFile('./my_module_bg_bg.wasm');
await init({ module_or_path: wasm });
```

**Browser (fetch):**

```typescript
import { init, MyModule } from './my_module.js';

const wasm = await fetch('./my_module_bg_bg.wasm');
const bytes = await wasm.arrayBuffer();
await init({ module_or_path: new Uint8Array(bytes) });
```

**Bundler (Vite, webpack, etc.):**

Import the `.wasm` file as an asset URL or arraybuffer depending on your bundler's configuration, then pass it to `init()`.

## Features

- All UniFFI primitives, strings, bytes, timestamps (`Date`), and durations
- Records as TypeScript `interface` types with optional field defaults
- Flat enums (string literal unions) and data-carrying enums (discriminated unions)
- Objects with constructors, methods, `free()` lifecycle, and `Symbol.dispose`
- Flat and rich error classes via `[Error]` and `[Throws]`
- Async functions and methods mapped to `Promise<T>`
- Callback interfaces as TypeScript `interface` declarations
- Trait interfaces with object return lifting
- Custom type aliases and external type imports
- Rename, exclude, and docstring (JSDoc) support
- Enum methods, constructors, and discriminant annotations
- Non-exhaustive enums and errors with catch-all variants
- Default argument values and optional parameters
